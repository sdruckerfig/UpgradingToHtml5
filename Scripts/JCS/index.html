HTML5 Fallbacks

Range

  • Plugin: jQueryTools Range Input
  • Will init on input['type','range'] (most plugins do not)
  • Has HTML5 freindly parameters like 'min' and 'max'
  • Has CDN

Date

  • Plugin: Keith Wood jQuery Datepicker
  • Allows for direct text input (many don't)
  • Recognizes input['type','date'] (most don't)
  • With help it can HTML5 freindly parameters like 'min' and 'max'
  • Plugin parameters would allow the reassignment of buttons to match Opera's calendar default.
  • Feature Detection Issue: Webkit registers support for "date" attribute.... but has lame support compared to Opera. Still working on this.

Validation

  • Plugin: nmcFormHelper
  • Has built-in fall back detection
  • Supports HTML5 attributes: placeholder, autofocus, required [with error handling], and pattern (regex validation).
  • Passes input vallues into URL
  • Other good option:
    • jQueryTools Validator
    • Disadvantages: has no placeholder support
    • Advantages: Has CDN, is more extendable.
  • Feature Detection Issue: Webkit registers that it has support for "required" attribute but does not support the error handling. As a result error handling is not availible for Webkit :-( Still working on this.

Cool feature of this fallback set: No images. Just code.
